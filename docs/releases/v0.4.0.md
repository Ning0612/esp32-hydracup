# HydraCup v0.4.0

## 主要更新：原生 FreeRTOS／ESP-IDF 遷移

- 韌體由 Arduino runtime 遷移至 PlatformIO `framework = espidf`，以 ESP-IDF 5.4 與 FreeRTOS 執行。
- `src/main.cpp` 只保留啟動與 loop orchestration；WiFi、HTTP、MQTT、Discord、NVS、LittleFS、HX711、蜂鳴器與 OLED 邏輯分拆至可重用模組。
- 控制流程集中於 FreeRTOS control task，MQTT、Discord、事件記錄與每日摘要使用背景 worker，避免網路或儲存 I/O 阻塞感測與飲水偵測。
- 保留既有 NVS 設定格式與資料分割區相容性；升級時不需要清除 NVS 或 `logfs`。

## 硬體與驅動

- OLED 已改用 ESP-IDF `i2c_master` bus/device API，移除 legacy I2C driver warning，保留 400 kHz 與內部 pull-up 設定。
- OLED 單位顯示統一為小寫 `g` 與 `ml`，避免字型與顯示截斷問題。
- 新增 `release-assets/`，提供 HydraCup 上下件的 STEP 與 STL CAD 檔案。

## WebUI、AP 與登入

- AP 設定頁恢復 WiFi 掃描、輸入欄位與操作按鈕，並與 WebUI 採用一致的視覺風格。
- 首次設定 AP SSID 為 `WaterCupTracker-Setup`，AP 密碼為 `12345678`。
- admin 沒有預設密碼；連上區域網路後前往 `/login`，首次設定至少 8 個字元的 admin 密碼。
- 修正登入 session cookie 保留與登入後跳轉，並保留 session、CSRF、逾時與登入失敗 rate limiting。

## 網路通知與穩定性修正

- Discord Webhook 改以 HTTP POST 發送，並回報 HTTP failure，避免訊息未送出時沒有明確診斷。
- 修正 WiFi scan 結果被重複消費造成 AP 掃描失效的問題。
- 保留既有 MQTT、Discord、NTP、事件記錄與 WebUI 設定資料，並完成 reconnect、heap、task stack 與 NVS 相容性驗證。

## 驗證

- `pio run -e esp32dev` — ESP-IDF firmware build 成功；RAM 使用 12.2%，Flash 使用 87.3%。
- `pio test -e native` — 16/16 測試通過。
- 已在 COM5 實機載入既有 NVS，成功連線 WiFi、MQTT 與 Discord；OLED 不再出現 legacy I2C warning。
- 已完成 24 小時穩定性測試，觀察 heap、task stack、WiFi reconnect 與既有 NVS 設定相容性。
