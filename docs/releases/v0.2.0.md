# HydraCup v0.2.0

## FreeRTOS runtime

- 新增 Core 1 `hydracup_control` task，集中管理 HX711、飲水偵測、提醒、蜂鳴器、OLED 與時間服務。
- 新增 `RuntimeCoordinator`，以一致 snapshot 與固定大小 command/result Queue 隔離 Dashboard 與控制 domain。
- tare 改為非阻塞取樣與 warm-up 流程；控制 task 期間不會被 HTTP handler 的 tare 操作卡住。
- Discord 改為持久 worker；MQTT 保留背景 worker；飲水 log 與 counter persistence 改為背景寫入。
- 新增 NVS `StorageLock`，並加入 counter version barrier、restore retry、跨日 settlement marker ack。
- `/api/status` 增加 RTOS heartbeat、健康狀態與 Queue drop telemetry。

## Validation

- `pio test -e native`：16/16 通過。
- `pio run -e esp32dev`：成功。
- firmware 與 `webfs` 已寫入 ESP32-D0WDQ6（COM5，MAC `cc:50:e3:8c:d9:d0`）。
- 實機曾成功輸出 RTOS heartbeat：`stack_free=6012`、`heap=191084`、`cmd_drop=0`、`result_drop=0`。

## Known limitations

- 硬體 cold boot 在 WiFi 連線階段曾觸發 brownout reset；需使用穩定 5V 供電後再進行完整 WiFi、Dashboard、MQTT、Discord 與 24 小時 soak 驗證。
- 尚未執行硬體上的 tare/calibrate、慢速 log client、WiFi reconnect 與長時間壓力測試。
